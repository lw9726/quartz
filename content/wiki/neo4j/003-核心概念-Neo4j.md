Neo4j 是一个无模式（schemaless）的图数据库。
> Neo4j is a schemaless database, so refactoring the data model is easy to do.

## 展开阐述
Neo4j 作为一个图数据库，其核心特点是“无模式”，这意味着在项目开发过程中，可以轻松地对[[002-核心概念-Graph Data Modeling|图数据模型]]进行重构和迭代。这一特性对于需要根据不断变化的业务需求（use cases）或为了提升查询性能而调整数据结构的项目来说至关重要。

在本文的[[012-核心流程-End-To-End Developer Flow|端到端开发流程]]中，Neo4j 是最终存储和查询图数据的平台。开发者可以通过[[009-核心实体-Claude Desktop]]等客户端，利用[[007-核心实体-Neo4j Data Modeling MCP server]]来设计数据模型，并使用[[008-核心实体-Neo4j Cypher MCP server]]来执行[[004-核心概念-Cypher]]查询，从而与本地的Neo4j实例（通过Neo4j Desktop或Aura）进行交互。

数据导入Neo4j的过程（[[014-核心流程-Data Ingestion Process]]）可以通过MCP服务器生成的[[004-核心概念-Cypher]]代码来完成，这些代码包含了创建索引、约束以及导入节点和关系的指令。
